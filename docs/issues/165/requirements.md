# Requirements: agents.mdの最適化

## User Stories

### 1. ベストプラクティスの抽出
**As a** 開発者
**I want to** 参考資料からAIコンテキスト定義（`agents.md`）のベストプラクティスを抽出する
**So that** プロジェクトのAI開発体験を向上させるための基礎知識を得ることができる

**Acceptance Criteria:**
- [ ] 指定されたQiita記事とGitHubリポジトリ（fastrender）が分析されていること
- [ ] AIにとって理解しやすい構造、情報の粒度、ディレクトリマップの記法などの重要ポイントが要約されていること

### 2. 現状分析とギャップ特定
**As a** 開発者
**I want to** 現状の `agents.md` を分析し、ベストプラクティスとのギャップを特定する
**So that** 具体的な改善点や不足している情報を明らかにできる

**Acceptance Criteria:**
- [ ] 現状の `agents.md` が読み込まれ、分析されていること
- [ ] ベストプラクティスと比較して、不足している要素や改善の余地がある箇所がリストアップされていること

### 3. セクションごとの改善と合意
**As a** 開発者
**I want to** トピック（ファイル構造、スタイル、アーキテクチャ等）ごとに改善案を議論し作成する
**So that** プロジェクト固有の事情を反映した最適なドキュメントを作成できる

**Acceptance Criteria:**
- [ ] 各セクション（ファイル構造、コーディング規約、アーキテクチャ概要など）について、具体的な記述案が提示されていること
- [ ] ユーザーとの議論を通じて、記述案がブラッシュアップされていること

### 4. agents.mdの最終化
**As a** 開発者
**I want to** 全ての改善を反映した完成版の `agents.md` を生成する
**So that** 今後の開発でAIがより正確かつ効率的にコンテキストを理解できるようになる

**Acceptance Criteria:**
- [ ] 合意された全ての内容を含む `agents.md` が作成されていること
- [ ] AIにとって読みやすく、プロジェクトの全体像を把握しやすい構造になっていること
