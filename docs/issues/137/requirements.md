# Requirements - Issue #137: 番組別パイプラインの実装と認証フローの確立

## ユーザーストーリー
1. **開発者/運用者として**、プライベートリポジトリで管理された設定ファイルを基に、特定の Spotify 番組に対して自動アップロードを実行したい。
2. **運用者として**、コンテナ内のブラウザ (VNC) で一度だけログイン操作を行えば、以降の自動実行ではその認証情報を利用してヘッドレスでアップロードが完了するようにしたい。

## 背景
これまでのステップで、セルフホストランナー（Docker）とオーケストレーター（private-ops）の準備が整った。本 Issue では、これらを組み合わせて実際に音源アップロードを行うロジックを実装し、実機での動作を検証する。

## 受け入れ基準 (Acceptance Criteria)

### 1. パイプライン実装
- [ ] `private-ops` リポジトリのワークフローが、指定された設定ファイル（JSON）から `Show ID` や `Base URL` などの必要なパラメータを読み出せること。
- [ ] 読み出したパラメータを引数として、`my-apps` 側の CLI ツール (`spotify-automation`) を正しく実行できること。

### 2. 認証フロー
- [ ] コンテナ内の VNC 経由でブラウザを操作し、Spotify の認証情報 (`.auth/spotify-auth.json`) を生成できる手順が確立されていること。
- [ ] 生成された認証情報を使用して、CLI ツールがヘッドレスモードでログイン状態を維持し、アップロード画面に遷移できること。

### 3. 実機検証
- [ ] テスト用番組 (`test-show.json`) に対して、実際の音声ファイル（ダミー可）のアップロードが成功すること。
- [ ] アップロードされたエピソードが Spotify for Podcasters のダッシュボード（または公開ページ）で確認できること。
