# Requirements Document - Issue 151

## 概要
`spotify-automation` プロジェクトの `test` ターゲットにおいて、標準出力および標準エラー出力が抑制されている設定を解除し、テストの実行結果（成功・失敗およびログ）が正しく Nx および CI 上で確認できるようにする。

## ユーザーストーリー

### US1: テストログの可視化
- **ユーザー**: 開発者
- **ストーリー**: テスト実行時に標準出力・標準エラー出力が表示されるようにしたい。
- **理由**: テストが失敗した際に、その原因をログから特定するため。

### US2: テスト失敗の正当な検知
- **ユーザー**: CIシステム / 開発者
- **ストーリー**: テストコマンドが失敗（終了コード非0）した場合、Nx のタスクも失敗として扱われるようにしたい。
- **理由**: テストが失敗しているにもかかわらず CI が通過してしまう（False Negative）を防ぐため。

## 受け入れ基準 (Acceptance Criteria)

- [ ] `apps/ui-automations/spotify-automation/project.json` の `test` ターゲットから `> /dev/null 2>&1` 等のリダイレクト記述が削除されていること。
- [ ] テスト実行時、コンソールにテストランナー（Jest/Vitest等）の出力が表示されること。
- [ ] テストが失敗した場合、`nx run spotify-automation:test` コマンドが非ゼロの終了コードを返すこと。
