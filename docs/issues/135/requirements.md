# Requirements - Issue #135: セルフホストランナー用 Dockerfile の作成と環境構築

## ユーザーストーリー

「開発者として、Spotify への自動アップロードを安全かつ自動化された環境で実行するために、専用のセルフホストランナー環境を利用したい。」

## 背景
パブリックリポジトリでのセルフホストランナー使用にはリスクが伴うため、隔離された Docker コンテナ環境を構築する。この環境は、認証情報の取得（手動操作）をサポートしつつ、実行時には最新のコードを動的に取得する堅牢な構成とする。

## 受け入れ基準 (Acceptance Criteria)

### 1. セルフホストランナー用 Docker イメージ
- [ ] 現行の開発環境 (`.devcontainer/Dockerfile`) と同様のツール群 (Ubuntu 20.04, Node.js, Python, TigerVNC, Fluxbox) がインストールされていること。
- [ ] GitHub Actions Runner Agent がインストールされ、設定スクリプトにより GitHub に登録可能な状態であること。
- [ ] noVNC が導入され、ブラウザからコンテナのデスクトップ環境を操作できること。

### 2. セキュリティと運用仕様
- [ ] ホストマシンのディレクトリをマウントすることなく、コンテナ単体で動作すること。
- [ ] コンテナ起動時に `my-apps` リポジトリを自動的に `git clone` し、環境構築 (`pnpm install` 等) を実行すること。
- [ ] 認証情報 (`spotify-auth.json`) は永続化せず、手動ログインによって一時的に取得・保持する運用が可能であること。

### 3. 動作検証
- [ ] セルフホストランナーとして GitHub Actions に認識されること。
- [ ] コンテナ内のブラウザを VNC 経由で操作し、Spotify へのログイン操作が可能であること。
- [ ] Playwright による E2E テストがコンテナ内で正常に動作すること。
