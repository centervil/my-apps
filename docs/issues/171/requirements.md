# 要件定義書 - Issue 171

## ユーザーストーリー
開発者として、複数のスラッシュコマンド（dev-start, task-start, review, dev-end, dev-cancel）を個別に覚えるのは面倒なので、単一の `/dev` コマンドから全ての工程を操作できるようにしたい。

## 目的
開発サイクルの各工程を `/dev` コマンドに統合し、Issue番号を引数として渡すことで、単一の入り口から全自動または個別の操作を行えるようにする。

## 受け入れ基準 (Acceptance Criteria)
- [ ] 統合コマンド `/dev` が設計・実装されていること。
- [ ] **全工程実行モード**: `/dev [Issue番号]` で、指定されたIssueの開発開始から終了までのワークフロー（開始 -> タスク化 -> レビュー -> 終了）が自動実行されること。
- [ ] **個別機能実行モード**: `/dev [サブコマンド] [Issue番号]`（例: `/dev review 123`）で、特定の工程のみが実行可能であること。
- [ ] 以下の既存個別コマンドファイルが削除されていること。
    - `dev-start.toml`
    - `task-start.toml`
    - `review.toml`
    - `dev-end.toml`
    - `dev-cancel.toml`
- [ ] 引数（サブコマンド、Issue番号）が正しく解釈され、各工程に適切なコンテキストが引き継がれること。
- [ ] サブコマンドとIssue番号の順序や組み合わせが柔軟に解釈されるプロンプト設計になっていること。
