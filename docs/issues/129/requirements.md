# Requirements - Issue 129

## 概要
現在、Spotify Automation の CLI に関するテストは、Playwright を使用した E2E テスト (`tests/e2e/cli.spec.ts`) と、独自に作成されたスクリプト (`scripts/test-cli.ts`) の両方に分散しています。これによりテストロジックが重複し、メンテナンス性が低下しています。本タスクでは、すべての CLI テストを Playwright に一本化し、コードの重複を排除します。

## ユーザーストーリー
開発者として、CLI の挙動（引数バリデーション、ドライラン、ファイル検索ロジック等）が単一のテストスイート（Playwright）で網羅されている状態にしたい。これにより、CLI の修正時に確認すべき場所が明確になり、テスト実行も効率化される。

## 受け入れ基準
- [ ] `scripts/test-cli.ts` で実行されていた以下のテストケースが `tests/e2e/cli.spec.ts` に移植されていること。
    - `showId` が欠落している場合に失敗すること。
    - `audioPath` が欠落している場合に失敗すること。
    - 存在しない `audioPath` が指定された場合に失敗すること。
    - 音声ファイルが含まれないディレクトリが `audioPath` に指定された場合に失敗すること。
    - 特定のファイルパスを `audioPath` に指定した場合に、そのファイルが正しく処理されること。
    - ディレクトリを `audioPath` に指定した場合に、その中の最新のファイルが自動的に選択されること。
- [ ] `scripts/test-cli.ts` およびその関連アセット（一時ファイル生成ロジック等）が削除されていること。
- [ ] `package.json` から `test:cli` などの不要になったスクリプトが削除されていること。
- [ ] `npx nx e2e spotify-automation` を実行した際、移植されたテストを含むすべてのテストが通過すること。
- [ ] テストで使用する一時的なオーディオファイルやディレクトリの生成・破棄が、Playwright のテストライフサイクル（`beforeAll`/`afterAll` 等）内で適切に管理されていること。
